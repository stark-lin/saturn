# Architecture Decisions

This directory is used to record architectural decisions that affect module boundaries, deployment methods, data models, permission models, or long-term technical trade-offs.

Rules:

```text
One decision per file
Filenames use NNNN-short-title.md
Record context, decision, consequences, and subsequent rollback conditions
Do not put ordinary implementation notes or to-do items into decisions
```
